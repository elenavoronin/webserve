A simple web server written in C++ that supports HTTP requests and serves static files. The following steps were undertaken to build the web server:

  • Project Initialization: Set up the project structure, including separate directories for source code and resources.
  • Implementing Socket Communication: Developed the core networking functionality to accept client connections and handle data transmission.
  • Request Parsing: Built a comprehensive HTTP request parser to interpret incoming requests accurately.
  • File Handling: Implemented logic to serve static files while ensuring proper error handling and security checks.
  • Client Handling: Integrated the ability to handle multiple clients using non-blocking I/O operations.
  • Error Logging: Established a logging mechanism to track errors and important events for easier debugging.
  • Testing: Conducted extensive testing to identify and fix bugs, ensuring the server operates as expected under various conditions.

Non-Blocking I/O Operations

  • Client Timeout: Ensuring the server could handle clients that take too long to respond without blocking the server's main operations.
  • Resource Management: Efficiently managing resources when dealing with multiple clients simultaneously.

Event-Driven Architecture

  • Managing Multiple Connections: Handling multiple I/O operations without creating a complex and inefficient multi-threading system.
  • Single Poll Mechanism: Using a single poll() system call to monitor all I/O events to simplify code complexity.

HTTP Protocol Compliance

  • Correct Status Codes: Ensuring the server responds with the correct HTTP status codes (200, 404, etc.) based on the outcome of client requests.
  • Method Handling: The server needed to support at least the HTTP methods GET, POST, and DELETE, adhering to web standards.

Static File Serving

  • Developed file handling logic to read and serve static files. The server uses open(), read(), and send() functions to retrieve files from the filesystem and send them to the client.
  • Implemented security measures to prevent unauthorized access to sensitive directories and files.

CGI Support with Forking

  • Integrated the fork() system call to create new processes for executing CGI scripts, allowing the server to run scripts independently of the main server process.
  • Established a mechanism to handle the output from these scripts and send it back to the client seamlessly.
